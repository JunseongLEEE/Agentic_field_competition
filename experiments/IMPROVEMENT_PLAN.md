# 데이터 레벨 역공학 분석 & 개선 설계 (2026-07-04, qwen_r16_ep1 val EM 0.6333 기준)

## 오답 구조 분해 (val 300, 오답 110건)

### 1. 오답의 42%는 인접 스왑
| 유형 | 건수 | 해석 |
|---|---|---|
| adj-swap (시간상 인접 2프레임 교환) | 46 (42%) | 나머지는 다 맞힘 — 마지막 한 쌍 구분 실패 |
| far-swap | 19 | |
| 3-pos/4-pos/reversal | 45 | 구조적 실패 (대부분 짧은 캡션) |

- pairwise 순서 정확도 85% vs EM 63% → 오답 대부분이 "거의 맞음"
- 스왑쌍 = 최고유사쌍인 비율 47.7% → 시각 유사성이 절반, 나머지는 캡션 미커버가 원인

### 2. 캡션 길이가 지배적 난이도 요인 (신규 발견)
| 캡션 길이 | n | EM |
|---|---|---|
| ≤15단어 | 47 | **0.191** |
| 16-25 | 51 | 0.490 |
| 26-35 | 162 | 0.759 |
| 36+ | 40 | **0.825** |

- 교차 분석: 짧은 캡션(≤20w)의 77%가 near-dup 프레임 → **"한 세그먼트 내 4프레임 + 단문 캡션"이 hard core** (train의 ~34%, EM ~0.21)
- 역공학 해석: 데이터는 두 종류 혼합 — (a) 전환 서술형 장문 캡션(재생성됨, 순서 정보 풍부), (b) 원본 ActivityNet풍 단문 캡션(순서 정보 없음, 시각 추론 필수)

### 3. Identity 캘리브레이션 버그 (원인 규명, 수정 배포)
- identity FN 17건 = 전체 오답의 15% (recall 67%, FP는 5건뿐)
- **원인**: uniform 셔플 증강이 identity 라벨을 1/24(4.2%)로 희석 ↔ test prior 15.5%
- **수정 1 (학습)**: `--id_prior 0.155` — 증강 시 identity 라벨 비율을 prior에 일치 ✅ 구현
- **수정 2 (추론)**: 24순열 스코어 저장(`--save_preds` 시 `*_scores.json`) → identity 로그보너스 b를 val 절반에서 튜닝, 나머지 절반 검증 ✅ 구현 (재학습 없이 적용 가능)
