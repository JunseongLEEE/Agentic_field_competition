# 심층 EDA — 학습 완료 후 계획 수립용 (2026-07-04)

분석 기반: train 9,535 전체, SigLIP2-so400m 임베딩 (`experiments/siglip_train.npz`)

## 발견 요약

### F1. 프레임 유사도가 매우 높다 — 난이도의 핵심
- 샘플 내 프레임 쌍 코사인 유사도 중앙값 **0.81**, 37.6%는 near-duplicate(>0.85)
- 난이도 버킷: distinct(<0.5) 0.5% / mid 18% / similar 44% / **near-dup 37.6%**
- near-dup 케이스는 같은 장면의 미세 변화(손 위치, 줌 정도)만으로 순서를 판별해야 함
- **시사점**: 입력 해상도가 성능 레버일 가능성 큼. 현재 MAX_PIXELS=256패치(이미지당)로 학습 중 → 추론/후속 학습에서 상향 실험 필요

### F2. 캡션 단서 구조
- "then" 60%, begins/starts 15%, zoom 17%, pan 9%, finally 10%
- 정규식 절 분할 결과: 1절 36.5% / 2절 47% / 3절 11% / 4절 5% → **4프레임에 1:1 대응하는 절 구조는 소수**. 대부분 캡션 절 수 < 프레임 수 → 같은 절 내 프레임들은 시각 단서로만 구분해야 함
- near-dup 샘플의 캡션은 카메라 단서 비율이 낮고(18% vs 48%), 객체/인물의 미세 동작 서술 중심

### F3. SigLIP 전역 임베딩의 순서 신호는 미약 (트랙 B 실패 원인 규명)
- 절-프레임 top-1 정렬 정확도 31.5% (chance 25%)
- 절 유사도 Hungarian 매칭 EM 9.3% < identity 15.5%
- **결론: 전역 의미 임베딩 + 표층 절 분할로는 이 과제를 풀 수 없음. joint VLM(트랙 A)이 유일한 경로**

### F4. 순열 분포는 순수 균등 (셔플 시)
- 셔플 샘플의 고정점 분포가 이론적 균등분포와 정확히 일치 (derangement 39.7% ≈ 9/23, 1고정 33.9% ≈ 8/23, 2고정 26.4% ≈ 6/23)
- 추가 구조·편향 없음 → 순열 prior 활용 여지 없음. 단 **identity(무셔플) 15.5% prior만 유효**
- No_ordering 여부는 캡션 통계(길이·접속사)로 전혀 구분 불가 — 시각·텍스트 모두 무신호(랜덤 서브셋)

### F5. train 내 동일 비디오 재사용 → CV 오염 존재
- 상호 최근접 & sim>0.98 쌍 542개 (같은 비디오의 다른 4프레임 샘플)
- 기존 val 954개 중 **96개가 train 쪽 near-dup 이웃 보유** → CV 소폭 낙관 가능성
- **조치 완료**: 오염 제거 클린 검증셋 `experiments/val_clean_ids.json` (858개, identity 참고치 0.1515) — 이후 모든 모델 평가는 val(954)와 val_clean(858) 병기

## 학습 완료 후 액션 플랜 (우선순위순)

1. **체크포인트 선택**: ep0/ep1 각각 val 300샘플 평가 (GPU 2장에 병렬 배분) → val_clean 기준 우수 쪽 선택
2. **오답 분석 (F1 연계)**: 오답을 난이도 버킷(sim_mean)별로 분해 → near-dup 집중 여부 확인. identity 샘플 재현율(15.5% prior 대비 과소/과잉 예측) 점검
3. **해상도 상향 실험 (F1)**: 추론 시 MAX_PIXELS 256→400+패치로 올려 val EM 변화 측정 (학습-추론 해상도 불일치 허용 범위 확인). 유의미하면 고해상도 재학습
4. **TTA (MERLOT)**: 입력 프레임 순서 2가지로 24순열 우도 평균 → val에서 이득 확인 후 적용
5. **제출 전략**: 오늘 남은 1회 = 첫 파인튜닝 모델(CV 근거 확보 시). 내일 2회 = 개선판(TTA/해상도). CV-LB 갭으로 합성 29% OOD 손실 추정 → 갭이 크면(>10%p) 과적합 의심, LoRA 강도/epoch 축소 재학습
6. **(여유 시) 실패 케이스용 보조 신호**: near-dup 케이스에서 캡션의 방향성 표현(zoom in/out, 이동 방향) 활용도가 낮으면 프롬프트에 "pay attention to camera distance/zoom, object positions" 힌트 추가 실험
